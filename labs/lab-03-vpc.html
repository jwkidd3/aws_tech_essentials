🌐 Lab 3

Create a Custom VPC and Deploy EC2

AWS Technical Essentials • Lab 3 of 6

Welcome to Lab 3. Students will create a custom VPC using the wizard and launch an EC2 instance inside it. This VPC will be used in Lab 6 for the load balancer and Auto Scaling Group.

📋 Lab Overview

🎯 Objectives

  • Create a VPC with public subnets using the wizard
  • Create a security group in the new VPC
  • Launch an EC2 instance in the custom VPC
  • Verify the web application works

📝 Lab Details

  • Duration: ~20 minutes
  • Region: Same as previous labs
  • Prerequisite: Lab 2 instance running
⚠️ Keep this VPC — it is needed for Lab 6.
This VPC and its subnets will be reused in Lab 6 for the ALB and Auto Scaling Group. Ensure all students have their Lab 2 instance running.

🔧 Create VPC — Settings

  1. Navigate to VPC in the console
  2. Click Create VPC
  3. Select "VPC and more"

The "VPC and more" wizard creates the VPC, subnets, internet gateway, and route tables in one step. Two public subnets in different AZs are required for the ALB in Lab 6. No private subnets or NAT gateways needed for this course.

🔧 Create VPC — Execute

  1. Review the Preview panel on the right
  2. Click Create VPC
  3. Wait for all resources to show green checkmarks
  4. Click View VPC
✅ Expected: VPC created with 2 public subnets, an internet gateway, and route tables — all auto-configured.
The wizard creates: VPC, 2 public subnets in different AZs, an internet gateway, and a route table with a 0.0.0.0/0 route to the IGW. This takes about 30 seconds.

🔒 Create Security Group

  1. In the VPC console left nav, click Security groups
  2. Click Create security group
  3. Name: employee-directory-app-sg
  4. Description: Allow HTTP access for Employee Directory
  5. VPC: Select employee-directory-vpc
TypePortSource
HTTP800.0.0.0/0
SSH22My IP
  1. Click Create security group
This security group must be in the new VPC (not the default VPC). Students often miss the VPC dropdown — make sure they select employee-directory-vpc. This SG will also be used by the Auto Scaling Group in Lab 6.

🚀 Launch EC2 in the New VPC

  1. Navigate to EC2 → Launch instance
  2. Name: employee-directory-app-vpc
  3. AMI: Amazon Linux 2023
  4. Instance type: t2.micro
  5. Key pair: employee-dir-key (from Lab 2)
  6. Under Network settings, click Edit:
    • VPC: employee-directory-vpc
    • Subnet: Choose any public subnet
    • Auto-assign public IP: Enable
    • Select existing security group: employee-directory-app-sg
Critical: students must select the new VPC, not the default. Auto-assign public IP must be enabled. If they select a subnet without checking auto-assign, the instance will not get a public IP.

🚀 Add User Data and Launch

  1. Under Advanced details, set IAM instance profile to EmployeeDirectoryAppRole (if available)
  2. Scroll to User data and paste:

  1. Click Launch instance
Same user data script as Lab 2. No placeholders or edits needed.

🌐 Test the Web Server

  1. Wait for the new instance to reach Running with 2/2 checks passed
  2. Copy the Public IPv4 address
  3. Open a browser tab: http://<PUBLIC_IP>
✅ Expected: The Employee Directory page loads — now running inside your custom VPC.
⏲ Note: Allow 1-2 minutes after "Running" for Apache to finish installing.
If the page does not load: verify using HTTP not HTTPS, check security group allows port 80, confirm a public subnet was selected, and that auto-assign public IP is enabled.

🛑 Stop the Lab 2 Instance

  1. In the EC2 console, find your original employee-directory-app instance (from Lab 2)
  2. Select it, click Instance state → Stop instance
💡 Why? The Lab 2 instance was in the default VPC. Now that you have a working instance in your custom VPC, you no longer need the old one running. Stopping saves cost.
Students should now have two instances: the old one (stopped, default VPC) and the new one (running, custom VPC). Do NOT terminate either instance yet.

⚠️ Resources to Keep

Do NOT delete these resources — they are needed for Lab 6:
  • VPC: employee-directory-vpc (with subnets, IGW, route tables)
  • Security Group: employee-directory-app-sg
  • Key Pair: employee-dir-key
  • IAM Role: EmployeeDirectoryAppRole (if created)

You may terminate the EC2 instance from this lab — Lab 6 will create new instances via Auto Scaling.

The VPC and its networking components are critical for Lab 6. Students can optionally terminate the EC2 instance since Lab 6 creates instances automatically through the Auto Scaling Group.

📚 Key Concepts

  • VPC = isolated network in the cloud
  • Public subnets have routes to an IGW
  • Multi-AZ enables high availability
  • Security groups are VPC-specific
Review key takeaways. The multi-AZ setup will be essential for Lab 6 when creating the ALB and Auto Scaling Group.

❓ Questions?

Ask your instructor before proceeding to the next module
